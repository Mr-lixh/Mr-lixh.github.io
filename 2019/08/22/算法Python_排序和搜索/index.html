搜索算法

搜索

搜索是指在一个集合中寻找特定的元素。返回布尔值或者元素位置。在 Python 中，我们可以使用 in 操作判断一个元素是否在集合中。

二分查找

在有序列表中，可以使用二分查找加快查找速度。二分查找首先从列表的中间位置开始搜索，如果相等，则结束查找，否则在可能含有该元素的一半列表中进行查找。下图表示了在一个有序列表中使用二分查找搜索54。

使用 Python 代码实现上述过程。

二分查找是分治算法的一个典型例子。我们可以使用递归调用实现该算法。

二分查找分析

二分查找的时间复杂度为 $O(\log n)$。尽管二分查找的时间复杂度比顺序查找要小，但是当数据规模 n 比较小时，额外的排序成本可能并不值得。事实上，我们需要考虑采取额外的排序工作来获得查找效益是否值得。如果排序一次可以多次查找，那么排序的成本就不那么重要了。

哈希

  • 哈希表是数据项的集合，其结构非常方便数据查询；
  • 哈希结构的查找复杂度为 $O(1)$；
  • 哈希表的每个位置称为 slot，存储一个数据项，并且使用从0开始的整数值命名；
  • 我们可以使用 Python 中的 list构造哈希表，使用 None 来初始化每一个元素值。
    下图初始化一个大小为 11 的哈希表。

数据项与槽的映射称为哈希函数。最常用的哈希函数是“余数方法”。通过对哈希表的大小进行取余，得到数据项的位置。下图表示在一个长度为 11 的哈希表中插入数据 54，26，93，17，77，31。其中有6个槽被占用。$\lambda = \frac {numberofitems}{tablesize}$ 称为负载因子。

当我们在哈希表中进行搜索时，只需要按照哈希函数计算出槽的名称即可，然后查看数据项是否存在。
这里我们只讨论了将数据项映射到哈希表中，不发生冲突的情况。实际上，我们经常会遇到不同的数据元素映射到同一个 slot 的情况，后面我们详细讨论这种情况。

哈希函数

  • 如果一个哈希函数能够把每个数据项映射到不同的槽内，该函数称为 完美哈希函数；
  • folding method方法将数据项切分成相等大小的片段（最后一个片段可能不同），将这些片段相加得到散列值，然后进行取余操作；
  • mid-square method方法将数据项平方后，提取结果数字的一部分作为散列值；

冲突解决

开放寻址法：
open addressing方法从原始哈希值的位置开始，以顺序的方式向后搜索，直到遇到第一个空槽。

  • 按顺序一次一个的查找空槽称为 linear probing。线性探测遇到元素聚集时，即在相同的散列值处发生许多冲突，导致周围的槽被冲突填满，对其他数据项产生影响。解决方法是使用一个增量来查询槽，使得冲突项分布的更均匀。
  • 解决冲突项的过程称为 rehashing。表达式为：$newhashvalue = rehash(oldhashvalue)$，$rehash(pos) = (pos + skip) % sizeoftable$。其中，skip 的大小必须使得 rehash 能够访问所有的槽，否则会导致哈希表的一部分不饿能被使用。为了确保这一点，哈希表的大小通常设定为素数；
  • quadratic probing 使用平方值作为增量；

链表法：
链表法 中，每个槽保存对相同散列值的数据项集合的引用。下图使用该方法解决散列冲突。

Map 抽象数据结构

Map 抽象数据类型是键值对的无序集合。如 Python 中的字典。其中的键是唯一的，因此键和值之间存在一对一的关系。其操作如下：

  • Map()：创建并返回一个空的 map 集合；
  • put(key, val)：添加一个键值对。如果键已经存在，则更新值；
  • get(key)：返回键 key 所对应的值；如果不存在，则返回 None；
  • del：删除键值对；
  • len()：返回键值对个数；
  • in：判断键 key 是否在 map 中。

排序算法

排序

  • 排序是将集合中的元素按照某一顺序进行重新排列；
  • 对于小的数据集，更倾向于使用简单的排序算法。对于大的数据集，则需要尽可能地优化算法来达到更高的效率；
  • 排序中主要发生的操作有 比较 和 交换。

冒泡排序

  • 冒泡排序通过比较相邻元素并交换无序的元素；
  • 每趟遍历都会将下一个最值放到最终的位置；
