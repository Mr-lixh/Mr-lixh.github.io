数据分析简介

数据分析的一般流程

  • 数据获取（抓取、日志）—>存储
  • 探索分析（有什么属性、值域分布、属性间的关联、从已有属性衍生出新属性）与可视化 （以上两步骤合起来为狭义的数据分析）
  • 数据预处理（数据格式转换）
  • 分析建模（分类、回归、聚类、关联、半监督模型）
  • 模型评估（从多模型中挑选最优模型）（以上称为数据建模与挖掘）

数据分析的含义与目标

数据分析：发现数据规律，并根据规律做出预测和决策。即利用统计分析方法，在数据中提取有用信息，并进行总结与概括的过程。

数据获取

数据获取手段

  • 数据仓库（如用户的浏览记录、购买记录）
    • 将所有业务数据汇总处理，构成数据仓库(DW)
      • 全部事实的记录
      • 部分维度与数据的整理（数据集市-DM）
    • 数据库 VS 数据仓库
      • 数据库面向业务存储，仓库面向主题存储（主题：较高层次上对分析对象数据的一个完整并且一致的描述）
      • 数据库针对应用(OLTP)，数据仓库面向分析(OLAP)
      • 数据库的组织规范，仓库可能有很大的冗余，相对变化大，数据量大
  • 监测与抓取
    • 抓取：直接解析网页、接口、文件的信息
      • 常用抓取工具：urllib、urllib2、requests、scrapy、PhantomJS、beautifulSoup、Xpath(lxml)
  • 填写、日志、埋点
    • 填写：用户填写的信息
    • 埋点：APP或网页埋点（特定流程的信息记录点，如访问、访客、停留时间、页面查看、跳出率，包括页面统计和行为统计）
  • 计算
    • 通过已有数据计算生成衍生数据

数据学习网站

  • 数据竞赛网站（Kaggle、天池）
  • 数据集网站（ImageNet/Open Images）
  • 各领域统计数据（统计局、机构、财报等）
