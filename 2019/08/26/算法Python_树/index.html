树

如图所示，Unix文件系统是一个树形结构。

树的定义一：树由一组节点和一组连接节点对的边组成。树具有以下属性：

  • 一个节点作为树的根节点；
  • 除了根节点，其他节点有且只有一个父节点；
  • 从根节点到每一个节点都有一条独一无二的路径；
  • 如果树中的每个节点最多有两个子节点，则为 二叉树。

树的定义二：树要么是空的，要么由一个根节点和零个或多个子树构成，每个子树也是树，且子树的根节点通过边连接到符树的根节点。

应用：解析树

如图所示，可以将数学表达式 ((7+3)∗(5−2)) 表示为一个解析树。

如何根据表达式的字符列表构造一个解析树？

  • 如果当前字符为 (，则为当前节点添加一个左孩子，并指向左孩子节点；
  • 如果为操作符，将当前节点的根节点的值设为操作符。并添加一个右孩子，同时指向右孩子节点；
  • 如果为操作数，将当前节点设为该数，指针重新指向父节点；
  • 如果当前字符为 )，指针指向父节点。

我们可以使用 getLeftChild 和 getRightChild 访问孩子节点，但如何访问当前节点的父节点呢？一种解决方法是使用堆栈。当我们要访问孩子节点时，将当前节点存入堆栈中。当要访问父节点时，只需要从堆栈中弹出一个元素即可。

优先队列和二叉堆

  • 在优先队列中，所有的元素按照优先级进行排序，优先级高的位于队列头部，优先级低的位于队列尾部；
  • 二叉堆的结构类似于树，但是实现时使用列表作为内部表示；
  • 二叉堆分为小顶堆和大顶堆。

小顶堆的定义

小顶堆的操作主要有以下几个：

  • BinaryHeap()：创建一个空的新二叉堆；
  • insert(k)：向堆中添加一个新的元素 k；
  • findMin()：返回具有最小键值的元素，不删除；
  • delMin()：返回具有最小键值的元素，并将其从堆中删除；
  • isEmpty()：判断堆是否为空；
  • size()：返回堆中元素的个数；
  • buildHeap(list)：使用一个键列表构造一个堆。

为了保证堆有 $O(logn)$ 的时间复杂度，我们需要使用 平衡二叉树 实现堆。可以通过构造 完全二叉树 来实现树的平衡。下图为一个完全二叉树。完全二叉树具有以下优点：

  • 可以使用列表实现完全二叉树，而不需要节点与引用或者嵌套列表；

二叉搜索树

我们可以使用 二叉搜索树 实现键值对的映射。搜索树的主要操作如下：

  • Map()：创建一个空 map；
  • put(key, val)：添加一个键值对。如果已经存在该键，则更新值；
  • get(key)：获取键所对应的值。不存在则返回 None；
  • del：使用 del map[key] 删除键值对；
  • len()：获取键值对的个数；
  • in：判断键是否在字典中。

二叉搜索树的特性：

  • 比父节点小的键节点都在左子树，比父节点大的键节点都在右子树。

一旦我们找到要删除的节点，需要考虑以下三种情况：

  • 要删除的节点没有孩子节点；
  • 要删除的节点只有一个孩子节点；
  • 要删除的节点有两个孩子节点。
  1. 第一种情况很简单，我们只需要删除该节点以及其父节点对该节点的引用即可。

  1. 第二种情况略微复杂。如果要删除的节点只有一个孩子节点，我们只需要将孩子节点替换掉当前节点即可。一共需要考虑六种情况，其中有左孩子或右孩子相对称。下面列出了当有左孩子时需要考虑的情况：
  • 如果当前节点是一个左孩子，那么更新其左孩子的父节点引用到当前节点的父节点，然后更新父节点的左孩子引用到当前节点的左孩子；
  • 如果当前节点是一个右孩子，那么更新其左孩子的父节点引用到当前节点的父节点，然后更新父节点的右孩子引用到当前节点的左孩子；
  • 如果当前节点没有父节点，则为根节点。只需要使用 replaceNodeData 更改根节点的属性即可。

  1. 第三种情况最为复杂。
