什么是反射？

JAVA 反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意方法和属性；这种状态获取信息以及动态调用对象方法的功能称为java语言的反射机制。

2.1 Java异常以及常用工具类体系

Error 和 Exception 的区别

  • Error：程序无法处理的系统错误，编译期不做检查；
    • NoClassDefFoundError：找不到 class 定义异常
    • StackOverflowError：深递归导致栈溢出
    • OutOfMemoryError：内存溢出
  • Exception：程序可以处理的异常，捕获后可能恢复。
    • RuntimeException：不可预知的，程序应当自行避免；
      • NullPointerException：空指针引用异常
      • ClassCastException：类型强制转换异常
      • IllegalArgumentException：传递非法参数异常
      • IndexOutOfBoundsException：下标越界异常
      • NumberFormatException：数字格式异常
    • 非 RuntimeException：可预知的，从编译期校验的异常，如 IOException、SQLException 等。
      • ClassNotFoundException：找不到指定 class 的异常
      • IOException：IO 操作异常

Java 的异常处理机制

  • 抛出异常：创建异常对象，交由运行时系统处理
  • 捕获异常：寻找合适的异常处理器处理异常，否则终止运行

Java 异常的处理原则

  • 具体明确：抛出的异常应能通过异常类名和 message 准确说明异常的类型和原因；
  • 提早抛出：应尽可能早的发现并抛出异常，便于精确定位问题；
  • 延迟捕获：异常的捕获和处理应尽可能延迟，让掌握更多信息的作用域来处理异常。

try-catch 性能比 if-else 语句低的原因

  • try-catch 影响了 JVM 的优化
  • 异常对象实例需要保存栈快照等信息，开销较大

2.2 Java 集合框架

List 和 Set 的区别

Map

HashMap、HashTable 和 ConcurrentHashMap 的区别

  • HashMap：线程不同步，效率高；延迟构造；可以插入空键
    • （java8以前）：数组+链表
    • （java8以后）：数组+链表+红黑树
  • 如何有效减少碰撞
    • 扰动函数：促使元素位置均匀分布，减少碰撞机会
    • 使用 final 对象，并采用合适的 equals() 和 hashCode() 方法，如使用 String、Integer 不可变对象，可以缓存hashCode，提高速度

  • HashTable：线程同步，只能串行，效率低；
  • 和 synchronizedMap 的区别是：锁对象不同
  • 如何优化 HashTable
    • 通过锁细粒度化，将整锁拆解成多个锁进行优化：只锁数组的一部分
    • 当前的 ConcurrentHashMap：CAS+synchronized使锁更细化

  • ConcurrentHashMap：不允许插入空键
    • 首先使用无锁操作 CAS 插入头结点，失败则循环重试
    • 若头节点已经存在，则尝试获取头节点的同步锁，再进行操作

三者的区别

  • HashMap 线程不安全，底层使用数组+链表+红黑树进行实现
  • HashTable 线程安全，锁住整个对象，数组+链表实现
  • ConcurrentHashMap线程安全，使用CAS+同步锁，数组+链表+红黑树
  • HashMap 的key和value均可以为null，而其他两个类不支持

J.U.C 知识点

  • java.util.concurrent：提供了并发编程的解决方案
    • CAS 是 java.util.concurrent.atomic 包的基础
    • AQS 是 java.util.concurrent.locks 包以及一些常用类如Semophore，ReentrantLock 等类的基础
  • J.U.C 包的类
    • 线程执行器 executor
    • 锁 locks
    • 原子变量类 atomic
    • 并发工具类 tools
    • 并发集合 collections

BlockingQueue

  • 阻塞队列：当队列空时，阻塞出队操作，队列满时，阻塞入队操作；
    • 主要用于生产者-消费者模式，在多线程场景时生产者线程在队尾添加元素，消费者线程在队头消费元素，通过这种方式达到将任务的生产和消费进行隔离的目的。
  • 主要的实现类：
    • ArrayBlockingQueue：数组构成的有界阻塞队列；
    • LinkedBlockingQueue：链表构成的有界/无界阻塞队列；
    • PriorityBlockingQueue：支持优先级排序的无界阻塞队列；
    • DelayQueue：一个使用优先级队列实现的无界阻塞队列，支持延迟获取；
    • SynchronousQueue：一个不存储元素的阻塞队列
    • LinkedTransferQueue
    • LinkedBlockingDeque：可以实现消费线程窃取其他工作队列（从尾部窃取，不发生冲突）

2.3 IO 机制

BIO、NIO、AIO

  • Block-IO：传统的java.io以及部分java.net包下的类。InputStream、OutputStream 和 Reader、Writer
    • 线程会一直阻塞

  • NonBlock-IO：构建多路复用、同步非阻塞的 IO 操作
    • Channels
    • Buffers
    • Selectors

  • Asynchronous IO：基于事件和回调机制，异步非阻塞
    • 基于回调：实现 CompletionHandler 接口，调用时触发回调函数
    • 返回 Future：通过 isDone() 查看是否准备好，通过 get() 等待返回数据

  • 三者对比
